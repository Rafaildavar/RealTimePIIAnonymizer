# RealTimePIIAnonymizer

Real-time masking of personal data in LLM requests for banking assistants.

## О проекте

В банке используются разные ИИ-ассистенты и агенты, которые помогают клиентам решать широкий круг задач: от консультаций по карточным продуктам до подбора инвестиционной стратегии или планирования поездок.

При обработке таких запросов могут использоваться персональные данные:
- ФИО
- номер телефона
- email
- паспортные данные
- номер банковской карты
- адрес проживания

Передача этих данных в LLM создает риски безопасности и юридические ограничения.

**RealTimePIIAnonymizer** — это прототип системы, которая в режиме реального времени:
- обнаруживает персональные данные в пользовательском запросе;
- маскирует их перед отправкой в LLM;
- сохраняет полезный контекст запроса;
- поддерживает безопасное восстановление данных в ответе.

## Что умеет система

- real-time обработка пользовательских запросов;
- mask/unmask pipeline;
- rule-based и heuristic detection;
- поддержка сущностей `NAME`, `PHONE`, `EMAIL`, `PASSPORT`, `CARD`, `ADDRESS`;
- безопасное восстановление ответа;
- оценка качества на размеченном датасете;
- интеграция ML-слоя на базе CRF для улучшения распознавания `NAME` и `ADDRESS`.

## Структура проекта

### `pii_realtime_cli.py`
Основной интерактивный скрипт для демонстрации работы системы в реальном времени.  
Принимает пользовательский запрос, находит персональные данные, маскирует их и позволяет проверить восстановление ответа LLM.

### `evaluate_dataset.py`
Скрипт для оценки качества алгоритма на размеченном датасете.  
Считает, насколько предсказание совпадает с эталоном, и сохраняет результаты анализа в отдельные файлы.

### `train_crf_pii.py`
Скрипт обучения CRF-модели на подготовленном датасете.  
Используется для добавления ML-слоя, который улучшает распознавание менее формализованных сущностей, например имен и адресов.

### `crf_recognizer.py`
Модуль с CRF-recognizer, который подключается в основной pipeline.  
Использует обученную модель `crf_pii_model.joblib` и возвращает найденные сущности в том же формате, что и rule-based детекторы.

### `crf_infer_pii.py`
Отдельный скрипт для проверки CRF-модели на новых строках.  
Нужен для быстрого тестирования ML-части без запуска всего основного пайплайна.

### `data_pers.csv`
Размеченный датасет для обучения и тестирования.  
Каждая строка содержит исходный текст и эталонный замаскированный вариант.

### `analysis_results.csv`
Файл с полными результатами проверки датасета:
- исходный текст,
- эталон,
- предсказание модели,
- совпадение или ошибка.

### `errors_only.csv`
Файл только с ошибочными примерами.  
Нужен для анализа слабых мест алгоритма и последующей доработки.

### `metrics.txt`
Краткая сводка по качеству:
- количество примеров;
- количество точных совпадений;
- accuracy;
- число ошибок.

### `crf_pii_model.joblib`
Сохраненная обученная CRF-модель.  
Используется в `crf_recognizer.py` и `crf_infer_pii.py`.

## Логика работы

1. Пользователь вводит запрос.
2. Система находит персональные данные.
3. Персональные данные заменяются на плейсхолдеры (`[NAME]`, `[PHONE]`, `[EMAIL]` и т.д.).
4. Замаскированный запрос может быть отправлен в LLM.
5. При необходимости ответ LLM проходит через безопасное восстановление данных.

## Используемые подходы

### Rule-based слой
Используется для хорошо формализованных сущностей:
- телефон
- email
- паспорт
- карта

### Heuristic слой
Используется для имен и адресов через шаблоны, контекст и словари.

### ML-слой
Используется CRF-модель для улучшения распознавания `NAME` и `ADDRESS`.
